# Документация к тестам

## test_api_basics.py
Содержит 10 основных тестовых сценариев:

### 1. test_get_request()
**Цель**: Проверить базовый GET запрос
**Проверки**: 
- Статус код 200
- URL в ответе соответствует запросу
- Структура ответа содержит обязательные поля

### 2. test_get_with_params()
**Цель**: Проверить передачу query-параметров
**Проверки**:
- Параметры корректно передаются
- Сервер возвращает параметры как строки
- Все переданные параметры присутствуют в ответе

### 3. test_post_request()
**Цель**: Проверить POST запрос с JSON
**Проверки**:
- Статус код 200
- Отправленные JSON данные возвращаются в ответе
- Структура ответа корректна

### 4. test_put_request()
**Цель**: Проверить PUT запрос (обновление)
**Проверки**:
- Статус код 200
- Данные для обновления корректно обрабатываются

### 5. test_delete_request()
**Цель**: Проверить DELETE запрос
**Проверки**:
- Статус код 200
- DELETE endpoint доступен

### 6. test_status_codes()
**Цель**: Проверить различные статус коды
**Проверки**:
- 200 OK (успех)
- 404 Not Found (не найдено)
- 500 Internal Server Error (ошибка сервера)
- 201 Created (ресурс создан)

### 7. test_response_headers()
**Цель**: Проверить передачу кастомных заголовков
**Проверки**:
- User-Agent передаётся корректно
- Accept заголовок обрабатывается
- Кастомные заголовки (X-*) проходят через сервер

### 8. test_basic_auth()
**Цель**: Проверить базовую аутентификацию
**Проверки**:
- С правильными credentials возвращается 200
- С неправильными credentials возвращается 401

### 9. test_response_structure()
**Цель**: Проверить структуру стандартного ответа
**Проверки**:
- Присутствуют обязательные поля: args, headers, origin, url
- Типы данных корректны

### 10. test_error_handling()
**Цель**: Проверить обработку ошибок
**Проверки**:
- Запрос к несуществующему endpoint возвращает 404
